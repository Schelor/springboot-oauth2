

##  feign-okhttp vs feign-java11 对比
```text
  ┌───────────────┬───────────────────────────────┬──────────────────────────────────────────────────────┐
  │ 维度           │ feign-java11 (JDK HttpClient) │                feign-okhttp (OkHttp)                 │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ 依赖           │ JDK 内置，无额外 jar            │ 引入 OkHttp (~1MB)                                   │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ 连接池         │ 基础支持                       │ 内置高效连接池，自动复用 TCP 连接                         │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ 超时控制        │ 较基础                         │ 精细：connect / read / write / call 超时分离           │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ HTTP 层拦截器  │ 无                             │ 支持 OkHttp 自己的 Interceptor（可独立打日志、重试）      │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ 生产成熟度      │ 较新，边角 case 较多            │ 业界标准，Android/服务端广泛验证                         │
  ├───────────────┼───────────────────────────────┼──────────────────────────────────────────────────────┤
  │ HTTP/2        │ 支持                           │ 支持                                                 │
  └───────────────┴───────────────────────────────┴──────────────────────────────────────────────────────┘


  结论：生产环境推荐 OkHttp。 连接池复用对 OAuth2 场景（每次请求都需要一次 token 校验 + 一次业务调用）收益明显，超时控制也更精细。

```
